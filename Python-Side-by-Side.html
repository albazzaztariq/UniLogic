Example 1 — append_to
Python
def append_to(element, to=None):
  if to is None:
    to = []
  to.append(element)
  return to
UL
function append_to(int element, none <to>)
  to.append(element)
  return to
end function

Python's mutable default argument to=[] is one of the language's most well-known traps. Default values are evaluated once at function definition time, not on each call. Every call that doesn't pass to shares the same list object — so appending in one call affects all future calls.

The standard workaround is to default to None and assign a new list inside the body. This works, but the intent is hidden behind a conditional that exists only to paper over the language behavior.

In UL, default parameter values are evaluated fresh on every call — there is no shared mutable state between invocations. The caller passes what it needs, or the function builds its own fresh state. No hidden shared objects, no defensive if to is None guards.
