AI/ML Practitioner Feedback

This project is being built with AI and ML workloads as a primary target. If you work in this space or play with its tools, your input on what's missing or what doesn't matter is worth its weight in gold. The list below covers what exists at the language and compiler level. Use the form at the bottom to say what matters, what's missing, and what's noise.

Language-Level Features

Mixed precision types
@dr precision = fp16 | bf16 | int8 | int4 declares computation precision at the file level, compiler emits appropriate instructions
Quantization as a type
q8 and q4 as first-class types, compiler manages dequantize/requantize at type boundaries automatically
Sparsity construct
@sparsity = auto on a type, compiler generates sparse compute paths when density warrants it
Tensor memory layout
@layout = column_major | row_major on types, compiler generates correct access patterns for cache efficiency
Alignment annotations
@alignment = 64 on types, guarantees 64-byte alignment for AVX-512
Static type system
All types known at compile time, enables every optimization decision before execution
Zero-overhead value types
type keyword produces C struct equivalent with no heap allocation, no vtable, no header overhead
Arena memory model
@dr memory = arena gives deterministic allocation with no GC pauses
No GIL equivalent
Compiled native code, no single-threaded execution bottleneck
Kernel fusion annotation
@fuse on a function body hints the compiler to generate a single fused operation
Vectorization guarantee
@dr vectorize = avx2 | avx512 | neon — compiler vectorizes or errors, no silent fallback
FMA guarantee
@dr fma = required — compiler guarantees fused multiply-add emission
Prefetch annotation
@prefetch(distance=auto) — compiler calculates optimal prefetch distance per target architecture
CUDA/ROCm/Metal targets
@dr compute = cuda | rocm | metal — same UL source compiles to GPU kernels
Inline assembly
@asm(x86_64) ... end asm for when you need direct hardware access with a UL fallback
Compile-time constants
const declarations substituted at compile time, zero runtime overhead
Static operation scheduling
@parallel on functions with independent branches, compiler generates static execution schedule
NUMA awareness
@dr numa = local forces memory allocation to stay on same socket as executing thread
Huge pages
@dr huge_pages = true reduces TLB misses for large model weight access
Thread affinity
@dr cpu_affinity = 0,1 pins threads to specific cores

Compiler-Level Features

LLVM backend
Full LLVM IR emission, all LLVM optimization passes available
AI workload LLVM flags
@dr compute = tensor automatically applies correct LLVM flags for AI workloads (disables aggressive unrolling, adds noalias on arrays, sets vectorization width)
Profile-guided compilation
--profile flag instruments code, profile reader produces summary, VS Code panels display flame graph and hot path highlighting
Normalization levels 0-3
Compiler can expand compound expressions, flatten call chains, split boolean expressions — controllable per file or per function
Cross-file DR conflict detection
Compiler reads DR settings of every file in workspace, warns at call sites that cross memory model or safety boundaries
Native binary via LLVM
Same source compiles to Windows, Linux, macOS, ARM via LLVM target triples
WebAssembly target
Compile to WASM for browser or server via WASI
Bytecode VM
Register-based bytecode interpreter for portable execution without native compilation
Profiler instrumentation
--profile injects timing and allocation hooks, produces NDJSON event log
DR behavior monitor
Real-time display of GC collections, refcount operations, arena resets during execution
