Deployment Targets

What is required beyond the transpiler to deploy UL code on each target platform.

Python Target

Transpiler + Python stdlib mapping + concurrency bridge + pip integration.

Python Standard Library Mapping
A UL standard library for Python interop — py.collections, py.io, py.os etc. that map to Python's stdlib. You write the UL call; the transpiler outputs the Python equivalent.
// UL
let data = io.read("file.txt")

// transpiles to Python
data = open("file.txt").read()
GIL-Aware Threading Model
Python's threading is constrained by the GIL. UL's concurrency primitives map to threading, multiprocessing, or asyncio depending on the use case — the transpiler selects the correct backend.
// CPU-bound → multiprocessing
@parallel
function process_batch(list items)
  // ...
end function

// I/O-bound → asyncio
@async
function fetch_all(list urls)
  // ...
end function
Runtime Shim
UL's refcount model differs from Python's. A small bridge handles object lifecycle differences so UL-transpiled code behaves correctly inside a Python runtime.
# pragma_rt.py — auto-imported

class ULRef:
  def __init__(self, obj):
    self._obj = obj
  def __del__(self):
    # lifecycle hook for UL DR
    pragma_release(self._obj)
Package Bridge
pip and PyPI are the ecosystem. UL declares Python dependencies in its module manifest and imports them as UL modules — the bridge resolves and installs them.
// pragma.module
target python
depend requests == "2.31.0"
depend numpy    >= "1.24"

// in UL source
import py.requests as http
let resp = http.get(url)

Java Target

Transpiler + JVM stdlib mapping + exception model bridge + Maven/Gradle integration.

JVM Standard Library Mapping
A UL standard library for JVM interop — jvm.collections, jvm.io, jvm.net etc. mapping to Java's stdlib. You write the UL call; the transpiler expands to the full Java boilerplate.
// UL
let resp = net.get(url)

// transpiles to Java
HttpClient client = HttpClient.newHttpClient();
HttpRequest req = HttpRequest.newBuilder()
  .uri(URI.create(url)).build();
var resp = client.send(req, ofString());
Type System Bridge
Java's type system is nominal and explicit. UL types map cleanly to JVM types, including handling Java generics which have no direct UL equivalent.
// UL type
type UserRecord
  string name
  int    age
end type

// JVM output
public record UserRecord(
  String name, int age
) implements ULType {}
Checked Exception Bridge
Java forces checked exceptions up the call stack. Transpiled UL code either wraps everything in RuntimeException or generates throws declarations from @throws annotations.
// UL
@throws(IOError)
function read_file(string path)

// JVM output
public static String readFile(String path)
    throws IOException {
  // ...
}
Build Integration
Maven and Gradle are the Java ecosystem. UL output slots into a standard Java build pipeline — the transpiler emits .java files into the expected source tree.
// pragma.module
target jvm
depend com.google.guava == "32.0"
depend org.slf4j        >= "2.0"

# src/main/pragma/ → src/main/java/
# then normal: ./gradlew build
