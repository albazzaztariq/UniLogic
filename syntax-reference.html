Pragma Language Reference

Standard Pragma. Source files use the .run extension. For lower level programming with memory management, pointers, and bitwise operations, click Base on the left.

Overview

Pragma is a statically-typed systems language that transpiles to C. It is designed to be readable to anyone who has written code in any modern language. The guiding rule: common operations use familiar lowercase syntax; anything that touches memory directly uses angle brackets < > as a visual signal.

Every program has at least a main function. Functions, objects, and constants are declared at file scope above it. This both acknowledges the older C language syntax and adds some clarity for beginners.

Program Structure

No wrapper blocks. File scope holds all declarations. main is the entry point.

// file scope
constant int MAX_SIZE = 100

function helper() returns void
  // ...
end function

function main()
  // entry point
end function
Note main or MAIN are both accepted as the entry point name.

Comments

// full line comment
int x = 10  // inline comment

Imports

import Math
import Collections

Collections

array

Fixed-size. Size is declared in the type. Zero-indexed.

array[int[4]] scores = [95, 87, 72, 100]
scores[0]          // 95

// Multi-dimensional: rows then columns
array[string[2][4]] grid
grid[0][2] = "hello"
Note array[int[]] is valid only when an initializer is present — size is inferred. Without an initializer, a size is required.

list

Dynamic, resizable. Optional capacity hint in brackets.

list[string[3]] names = ["Alice", "Bob", "Carol"]
names.add("Dave")
names[1]           // "Bob"

list[int[]] nums   // empty list, no capacity hint

map

[] for unbounded dynamic. [n] for initial capacity hint.

map[string, int][] wordCount
map[string, float][10] scores

Casting

Use the type name as a call. Float-to-int truncates — no rounding.

float f = 3.9
int n   = int(f)          // 3
string s = string(42)    // "42"
float g  = float(10)     // 10.0
bool b   = bool(1)       // true

If / Else

if score > 200
  print "high score"
else if score > 100
  print "good score"
else
  print "keep trying"
end if

While

int count = 0
while count < 5 do
  count++
  print count
end do

For

Two forms — use whichever reads better.

// expressive
for int i = 0 while i < 10 do i++
  print i
end for

// C-style
for i=0, i<10, i++
  print i
end for

For Each

list[string[3]] names = ["Alice", "Bob", "Carol"]

for each name in names
  print name
end for

Match — no fall-through

Colon on same line for single statements. Colon then newline for multi-statement cases.

int code = 2

match code
  1: print "one"
  2: print "two"
  3:
    print "three"
    print "still three"
  default: print "other"
end match

Switch — fall-through

Execution continues into the next case unless escape stops it.

switch code
  1:
  2:
    print "one or two"
    escape
  3:
    print "three"
    escape
end switch

Try / Catch

Multiple exception types in one catch use or.

  // risky code

Strings

string is a first-class type in Pragma. There is no char* or null-terminated array at this level — the runtime handles it. Use string(x) to convert any value to a string.

Base Pragma Reference

Low-level operations and systems work. These concepts may not exist or be as easily accessible in Python, Java, or other modern languages. The angle brackets < > are the signal that you are working at the memory level. Base Pragma files use the .base extension to denote difference.

Why Base Pragma?

Pointers

A pointer holds a memory address. Declared as type pointer name. The type tells the compiler how many bytes to read at that address and how to interpret them.

int x = 42
int pointer p = address<x>      // p holds the address of x

int8 b = 7
int8 pointer bp = address<b>    // 8-bit pointer
The angle bracket rule Any operation involving a raw memory address uses < >. This is the consistent visual signal across all of Base Pragma — not capitalisation, not special syntax, just the brackets.

address and ref

PragmaC equivalentMeaning
address<x>&xTake the address of variable x
ref<p>*pRead the value at the address in p
int val = 99
int pointer p = address<val>
int copy = ref<p>             // copy = 99
ref<p> = 100                  // write through pointer — val is now 100

mem — Raw Address Access

Read or write at any numeric address. Used for memory-mapped hardware registers.

uint32 pointer reg = address<someVar>
mem<reg>                          // read at address reg
mem<0xDEAD0000>                  // read at hard-coded address
mem<0xDEAD0000> = 1              // write to that address

Pointer Arithmetic

Offset into a pointer using angle brackets. Offset units are the pointer's element type — not raw bytes.

int pointer p = address<arr>
p<0>          // element 0 — same as ref<p>
p<5>          // element 5 — like *(p+5) in C
p<i + 2>      // element i+2
p<+3>         // advance 3 from current position

size

Byte size of a type. Equivalent to sizeof in C.

int s1 = size<int64>    // 8
int s2 = size<float>     // 4
int s3 = size<uint8>    // 1

nothing pointer — Generic Pointer

A pointer with no specific type. Points to "nothing in particular." Equivalent to void* in C. Use when a function must accept or return a pointer of any type.

nothing pointer generic = address<anyVar>

// Assign to a typed pointer — the declaration carries the type:
int pointer typed = generic

Bitwise Operations

Operate at the individual bit level. Use uint types for bit manipulation — applying these to signed integers flips the sign bit and produces negative results.

PragmaCMeaning
both1&Bitwise AND — output bit is 1 only if both input bits are 1
both0~(a|b)Bitwise NOR — output bit is 1 only if both input bits are 0
either1|Bitwise OR — output bit is 1 if either input bit is 1
delta^Bitwise XOR — output bit is 1 if the bits differ
bitflip x~xBitwise NOT — inverts every bit
negate x-xArithmetic negation
x left nx << nLeft shift — multiply by 2ⁿ
x right nx >> nRight shift — divide by 2ⁿ (logical on uint)
left=<<=Left shift assign
right=>>=Right shift assign
uint32 flags = 0
flags = flags either1 4      // set bit 2
flags = flags both1 0xFF     // mask to low byte
flags = flags delta 1         // toggle bit 0
flags = flags left 2          // shift left 2 positions
flags = flags right 1         // shift right 1 position
flags = bitflip flags          // invert all bits
Use uint types for bit manipulation bitflip on a signed int flips the sign bit. bitflip 5 gives -6, not 250. Always use uint8, uint16, uint32, or uint64 when working with bits.

Bit Fields

Declare a field with a specific number of bits using angle brackets after the name. Fields pack into adjacent bits in declaration order. Group them in an object to control struct layout.

object PackedHeader
  uint is_valid <1>    // 1 bit
  uint count    <7>    // 7 bits — total so far: 1 byte
  uint32 value         // 4 bytes, no bit constraint
end object
Note Bit field groups should total to a byte boundary (8, 16, 32, or 64 bits) for predictable cross-platform layout.

pack() — Object Packing

By default the compiler may insert padding between fields to align them in memory. pack() removes all padding — every field follows immediately after the previous one with zero gaps. Required for network packet headers and file format objects where byte position must be exact.

pack(PackedHeader)
Warning Packed objects can cause misaligned memory access on some processor architectures, triggering a hardware fault or a silent performance penalty. Only use when exact byte layout is required.

Why Base Pragma?

Not every program needs to manage memory, work with pointers, or think in bits. Most application code never touches any of this. Separating it into its own file type means beginners and those who don't need these details are never forced to encounter them. A .run file stays clean. A .base file signals immediately that something lower-level is happening here. That separation makes programming more comfortable for people who aren't ready for it, and clearer for those who are.

Coming from C

If you already know C, Base Pragma maps directly onto what you know. The semantics are the same — only the syntax changes. The goal is maximum readability without sacrificing any low-level control.

object vs struct In C, a struct is a named group of typed fields — nothing more. A Pragma object is the same thing, with the option to add methods, inheritance, interfaces, default field values, and constructors. If you ignore those features and just declare fields, it compiles to an identical struct. There is no separate struct keyword in Pragma because the only difference is capability, not concept.
