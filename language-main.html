UniLogic Language Reference

UL is divided between Standard, Base, and Mods. It's one language and syntax, but Base was separated for work closer to hardware, while Mods is more of interest to more sophisticated development and enterprises. Buttons on the left toolpane allow navigation between the three syntax references.

Program Structure

C Differences ↓

Every UL program has at least a main function. Functions, types, and constants are declared at file scope above it. This both acknowledges the older C language syntax and adds some clarity for beginners.

No wrapper blocks. File scope holds all declarations. main is the entry point.

function main() returns int
  // entry point
end function

Comments

// full line comment
int x = 10  // inline comment

Imports

import Math
import Collections

String

A first-class type. The underlying char array size is hidden — visible on hover in the IDE (LSP). When compiling to C, the compiler performs flow analysis to determine the maximum size the string can reach across the program. For runtime inputs, allocation is bounded to a safe maximum.

string name     = "hello"
string greeting = "Hi " + name
Note Memory safety cannot be guaranteed for runtime string inputs (user input, file reads). The compiler will warn when safety: checked is set.

Variables

int x = 10
float pi = 3.14159
string msg = "hello"
bool active = true
complex z = complex(3, 4)   // 3 + 4i
int n                       // defaults to 0

Note: types must be declared for variables unless types: dynamic is set in the Dynamic Runtime configuration.

Assignment:

x = 42
x += 5    x -= 2    x *= 3    x /= 4
x++        x--

Constants

constant int MAX_SPEED = 100
constant string APP_NAME = "Pragma"

Collections

Array

Fixed-size. Size declared after the type keyword. Zero-indexed.

array int[5] a = [1,2,3,4,5]
a[2]              // 3

array int a       // no declared size — decays to pointer

// Multi-dimensional: rows then columns
array int[5][3] grid
grid[0][2] = 42

List

Dynamic, resizable. Capacity hint in brackets.

list string[3] names
list string[3][2] grid   // 2D list

Map

Key-value store with optional capacity hint.

map [string,int][50] wc   // string→int, capacity 50

wc["key"]           // read  (calls wc_get)
wc["key"] = val     // write (calls wc_set)
wc_has("key")       // 1 if key exists, 0 otherwise
wc_remove("key")   // delete a key

Casting

Use change(expr)->Type. Float-to-int truncates — no rounding.

int n    = change(3.9)->int      // 3 — truncates
string s = change(42)->string    // "42"
float g  = change(10)->float     // 10.0
bool b   = change(1)->bool       // true

If / Else

if score > 200
  print "high score"
else if score > 100
  print "good score"
else
  print "keep trying"
end if

While

int count = 0
while count < 5 do
  count++
  print count
end do

Do / While

Executes the body at least once before checking the condition.

do
  print count
  count++
while count < 5 end do

For

Two forms — use whichever reads better.

// expressive
for int i = 0 while i < 10 do i++
  print i
end for

// C-style
for i=0, i<10, i++
  print i
end for

For Each

list string[3] names = ["Alice", "Bob", "Carol"]

for each name in names
  print name
end for

Match — no fall-through

Colon on same line for single statements. Colon then newline for multi-statement cases.

int code = 2

match code
  1: print "one"
  2: print "two"
  3: print "three"
  default: print "other"
end match

Iterate — fall-through

Execution continues into the next case unless escape stops it.

iterate code
  1: print "one"
  2: print "two"
     escape
  3: print "three"
     escape
end iterate
Coming from C iterate is C's switch. Fall-through by default — identical behaviour. escape is break.

Loop Control

teleport / portal

portal declares a label. teleport jumps to it unconditionally. Both must be in the same function scope.

portal retry

// ... code that may need to restart ...

if failed
  teleport retry
end if

Functions

function add(int a, int b) returns int
  return a + b
end function

function greet(string name)
  print "Hello, " + name
end function

function factorial(int n) returns int
  if n <= 1
    return 1
  end if
  return n * factorial(n - 1)
end function

Note: if a return type is anything other than nothing, it must be declared.

Types

type Player
  string name
  int age
  float score = 0.0
end type

type ProPlayer inherits Player
  int rank
end type

Player p = new Player["Alex", 25, 98.5]
print p.name
Coming from C type is a struct. Plain field declarations compile to an identical memory layout. Methods and inheritance are additive — ignore them and a type is exactly a struct.

Fixed — Enumerations

Named integer constants. Access as Name.member.

fixed Priority
  low    = 1
  medium = 2
  high   = 3
end fixed

int p = Priority.high    // p = 3
Coming from C fixed is enum. Access is dot notation — Priority.high instead of the typical PRIORITY_HIGH convention.

Print

print "hello world"
print "Score: " + change(score)->string
print add(3, 7)

prompt(msg) prints msg, reads a line from stdin, and returns a string. C and Python targets only.

string name = prompt("Enter your name: ")

Result / Option

UL has no exception syntax. There is no try, catch, throw, or raise. Errors are values. The standard library provides Result and Option — tagged union types that make error handling explicit and compiler-enforced.

Result

Result has two variants: ok wrapping a success value, and error wrapping an error value with type information. You cannot use the success value without first unwrapping it — the compiler rejects code that ignores the error case.

// a function that can fail returns Result
function read_file(string path) returns Result<string, IOError>
  // ...
end function

// explicit match on both variants
match read_file("data.txt")
  ok(contents)  →  print contents
  error(e)        →  print "failed: " + e.message
end match

Propagation operator

Appending ? to a call means: if the result is error, return that error from the current function immediately; if it is ok, unwrap the value and continue. The current function must itself return a compatible Result type.

function process(string path) returns Result<int, IOError>
  string contents = read_file(path)?   // propagates error, unwraps ok
  return ok(contents.length())
end function

Option

Option represents the possible absence of a value. some wraps a value; none represents absence. Use it anywhere a value may legitimately not exist — avoid returning null or a sentinel.

function find_user(int id) returns Option<User>
  // ...
end function

match find_user(42)
  some(user)  →  print user.name
  none        →  print "not found"
end match

Transpilation boundary

When generating Python or JavaScript, the compiler wraps target-runtime calls in try/except or try/catch at the boundary and converts caught exceptions into error variants before they surface in UL code. The UL developer always sees Result. Exceptions from the target runtime are an implementation detail of the generated code.

When XPile transpiles foreign code into UL, each source language's error mechanism is normalised: Python exceptions map to error variants with the exception type preserved; C error codes and null returns map to error variants; JavaScript promise rejections map to error in async contexts.

killswitch

A runtime assertion that terminates the program immediately if the condition is false. Active when safety: checked. Compiled out entirely when safety: unchecked — zero overhead in production builds.

killswitch index < length          // terminates if index >= length
killswitch ptr != null             // terminates if ptr is null
killswitch result == ok            // terminates if result is not ok

@Mods

Compiler modifiers prefixed with @. Placed on the line before a declaration, they extend its meaning without replacing any keywords.

Concurrency
@async / @await
Marks a function as a coroutine. The function may suspend at any @await point and resume without blocking the calling thread. Requires concurrency: async in the DR block. Maps to JS Promises, Python asyncio, or JVM virtual threads depending on target.
@async
function load_dashboard()
  let user    = @await fetch("/api/user")
  let reports = @await fetch("/api/reports")
  return Dashboard(user, reports)
end function
Targets
@platform
Restricts a function to one or more compile targets. When multiple @platform overrides exist for the same function name, the compiler selects the correct one. The undecorated declaration acts as the shared signature.
function format_date(int ts) string   // shared signature

@platform(c)
function format_date(int ts)
  return strftime("%Y-%m-%d", ts)
end function

@platform(js)
function format_date(int ts)
  return js_date(ts).toISOString()
end function
Safety
@nullable
Marks a field or parameter as explicitly nullable. Without this modifier, the compiler assumes non-null and enforces it. Absence of @nullable is the non-null contract.
type Session
  string            token
  @nullable string  user
  @nullable int     expiresAt
end type
Types
@data
Generates structural equality, hashing, and a copy() method. Two instances are equal if all fields match by value. Without @data, equality is by reference.
@data
type Point
  int x, y
end type

let a = Point(1, 2)
let b = a.copy(y = 5)   // Point(1, 5)
@sealed
Prevents a type from being extended outside the module it is declared in. The compiler knows every possible subtype, enabling exhaustive match checks.
@sealed
type Shape
  string kind
end type

type Circle extends Shape
  int radius
end type
@singleton
Exactly one instance ever exists. The runtime creates it on first access. No constructor is callable externally.
@singleton
type AppConfig
  string env
  int    maxConnections
end type

AppConfig.env = "production"
@alias
Declares a named alias for an existing type. Fully interchangeable with the original — adds meaning without wrapping or allocating.
@alias UserId       = int
@alias EmailAddress = string

function find_user(UserId id)
  // id is an int — no overhead
end function
Functions
@callembed
Substitutes the function body at every call site at compile time. No call overhead. Enforced — the compiler will not ignore it.
@callembed
function clamp(int v, int lo, int hi)
  if v < lo
    return lo
  else if v > hi
    return hi
  else
    return v
  end if
end function
@extends
Makes a free function callable as a method on the named type. The type itself is not modified — resolved entirely by the compiler at the call site.
@extends(string)
function truncate(string s, int max)
  if s.length <= max
    return s
  end if
  return s.slice(0, max) + "…"
end function

let t = title.truncate(40)
@operator
Maps a named function to an operator symbol for a type. Supported: + - * / % == != < > <= >=.
@operator(+)
function add(Vector2 a, Vector2 b)
  return Vector2(a.x + b.x, a.y + b.y)
end function

let v = origin + offset
@receiver
The function executes in the context of the named type — its fields are in scope without qualification. Foundation for DSL builders.
@receiver(QueryBuilder)
function build_query()
  from("users")
  where(active = true)
  limit(100)
end function
@delegate
Forwards all interface calls on a type to a named field of that interface type. The compiler generates every forwarding method — no boilerplate required.
type TimestampedLogger
  @delegate Logger inner    // log() forwarded to inner.log() automatically
  string           format
end type

Base UL Reference

Low-level operations and systems work. These concepts may not exist or be as easily accessible in Python, Java, or other modern languages. The angle brackets < > are the signal that you are working at the memory level.

Not every program needs to manage memory, work with pointers, or think in bits. Most application code never touches any of this. If that describes your work, the Standard page is everything you need — you have no reason to be here. This section exists for systems-level work: memory management, pointer arithmetic, bitwise operations.

Pointers

A pointer holds a memory address. Declared as type <name>. The type tells the compiler how many bytes to read at that address and how to interpret them.

int x = 42
int <p> = address(x)      // p holds the address of x

int8 b = 7
int8 <bp> = address(b)    // 8-bit pointer
The angle bracket rule Any operation involving a raw memory address uses < >. This is the consistent visual signal across all of Base UL — not capitalisation, not special syntax, just the brackets.
Coming from C int <p> = address(x) is int *p = &x in C. The angle brackets wrap the pointer name, and address() replaces &.

address and deref

ULC equivalentMeaning
address(x)&xTake the address of plain variable x
address(<p>)&pTake the address of pointer p
deref(<p>)*pRead the value at the address in p
int val = 99
int <p> = address(val)
int copy = deref(<p>)             // copy = 99
deref(<p>) = 100                  // write through pointer — val is now 100
Coming from C deref(<p>) is *p in C. Reading: int copy = deref(<p>) is int copy = *p. Writing: deref(<p>) = 100 is *p = 100.

Raw Address Access

Read or write at any numeric address. Used for memory-mapped hardware registers.

<reg>                    // read at address reg
<0xDEAD0000>             // read at hard-coded address
<0xDEAD0000> = 1         // write to that address

nocache

Marks a memory-mapped variable as volatile — every read goes directly to hardware, bypassing the CPU cache. Use for hardware registers whose value changes outside program control.

nocache uint32 sensor = <0xDEAD0000>   // read from register, never cached
<0xDEAD0000> = sensor + 1               // write back

Pointer Arithmetic

Use <p> + n to advance a pointer by n elements. Units are the pointer's element type — not raw bytes.

int <p> = address(arr)
<p>[0]        // element 0 — same as deref(<p>)
<p> + 5        // element 5 — like *(p+5) in C
<p> + (i + 2)  // element i+2
<p> + 3        // advance 3 from current position

size

Byte size of a type. Equivalent to sizeof in C.

int s1 = size(int64)    // 8
int s2 = size(float)     // 4
int s3 = size(uint8)    // 1

none — Generic Pointer

A pointer with no specific type. Equivalent to void* in C. Use when a function must accept or return a pointer of any type.

none <generic> = address(anyVar)

// Assign to a typed pointer — the declaration carries the type:
int <typed> = generic

Arena

An arena is an allocator object. Create it, allocate from it, free the entire thing at once. All allocations are reclaimed when the arena is freed — no individual frees needed.

arena a  = arena.new()
none <buf> = a.alloc(size(int) * 64)  // allocate 64 ints from the arena
// use buf ...
a.free()                               // entire arena released at once

Arena implementations are in the standard library or can be user-defined. Pass the arena explicitly to any function that needs to allocate from it.

memtake / memgive

Raw allocation in manual memory mode. memtake requests a block of bytes from the system allocator. memgive returns it. Use when arena allocation is not appropriate and you need direct control over individual allocations.

none <buf> = memtake(size(int) * 32)   // allocate 32 ints
// use buf ...
memgive(buf)                              // release the block

Bitwise Operations

Operate at the individual bit level. Use uint types for bit manipulation — applying these to signed integers flips the sign bit and produces negative results.

ULCMeaning
both1&Bitwise AND — output bit is 1 only if both input bits are 1
both0~(a|b)Bitwise NOR — output bit is 1 only if both input bits are 0
either1|Bitwise OR — output bit is 1 if either input bit is 1
delta^Bitwise XOR — output bit is 1 if the bits differ
bitflip x~xBitwise NOT — inverts every bit
negate x-xArithmetic negation
x left nx << nLeft shift — multiply by 2ⁿ
x right nx >> nRight shift — divide by 2ⁿ (logical on uint)
left=<<=Left shift assign
right=>>=Right shift assign
uint32 flags = 0
flags = flags either1 4      // set bit 2
flags = flags both1 0xFF     // mask to low byte
flags = flags delta 1         // toggle bit 0
flags = flags left 2          // shift left 2 positions
flags = flags right 1         // shift right 1 position
flags = bitflip flags          // invert all bits
Use uint types for bit manipulation bitflip on a signed int flips the sign bit. bitflip 5 gives -6, not 250. Always use uint8, uint16, uint32, or uint64 when working with bits.

Bit Fields

Declare a field with a specific number of bits using type.bits=N fieldname syntax. Fields pack into adjacent bits in declaration order.

type PackedHeader
  uint.bits=1 is_valid    // 1 bit
  uint.bits=7 count       // 7 bits — total so far: 1 byte
  uint32 value            // 4 bytes, no bit constraint
end type
Note Bit field groups should total to a byte boundary (8, 16, 32, or 64 bits) for predictable cross-platform layout.

pack() — Type Packing

By default the compiler may insert padding between fields to align them in memory. pack() removes all padding — every field follows immediately after the previous one with zero gaps. Required for network packet headers and file format types where byte position must be exact.

pack(PackedHeader)
Warning Packed types can cause misaligned memory access on some processor architectures, triggering a hardware fault or a silent performance penalty. Only use when exact byte layout is required.
Coming from C pack(TypeName) is __attribute__((packed)) or #pragma pack(1) in C. Same effect — removes struct padding for exact byte layout.

Maps directly to C's char[]. Size, allocation, and freeing are your responsibility.

type vs struct In C, a struct is a named group of typed fields — nothing more. A UL type is the same thing, with the option to add methods, inheritance, default field values, and constructors. If you ignore those features and just declare fields, it compiles to an identical struct. There is no separate struct keyword in UL because the only difference is capability, not concept.

Different languages behave differently.

Rust already toggles safety with unsafe blocks. Inside unsafe, you get raw pointers, manual memory, no borrow checker. Outside, you get full safety guarantees. Same language, two modes, toggled per-block.

C# has unsafe context too — lets you use pointers and fixed-size buffers, opting out of GC for that scope.

D has @nogc annotation — marks functions that aren't allowed to allocate from the garbage collector. You can mix GC and manual memory in the same program.

Zig lets you choose allocators per-call. Want arena allocation here, general purpose there, no allocation at all in this hot path? You pass the allocator explicitly.

Swift has both ARC (reference counting) and unsafe pointers available in the same language.

But no language puts all of these on a single dial panel. Every one picks a default and lets you escape from it in limited ways. UL has no default — you declare your policy and the compiler enforces it.
memory: manual // or gc, or refcount, or arena safety: checked // or unchecked types: strict // or dynamic int_width: 32 // or 64, or platform concurrency: threads // or async, or cooperative

Every function that allocates, every data structure, every IO operation works under all configurations. A linked list under GC doesn't need a destructor. Under manual memory it does. Under refcount it needs weak reference support.

One standard library with three implementations of everything makes this a reality.

There's no novel ideas, no reinventing the wheel: it allows anyone to write in C without the more unintuitive syntax of C, as well as understand other C programs directly in their language. If adopted more widely, it means everyone can understand directly each other's programs because syntactical style and rules are either expressed or familiar. And with the new heavier ML and AI workloads, it means hot paths and runtime become first-class citizens.
