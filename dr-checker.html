DR Conflict Checker

Check whether two files' Dynamic Runtime settings are compatible before linking them.

Each .ul file can declare its own DR configuration. When two files interact (one calls a function in the other), their settings must be compatible. This tool checks a pair of configurations against the full conflict matrix.

Conflict Rules

The checker evaluates within each file (are the file's own settings self-consistent?) and across files (can File A and File B interoperate?). Incompatible means the combination will not compile. Warning means it will compile but may cause runtime issues.
