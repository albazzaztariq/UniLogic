Enterprise Annotations

Proposed @ annotations for large-scale, multi-target, and enterprise application development.
Type Declarations
@data
Generates structural equality, hashing, and a copy() method. Fields are compared by value, not by reference.
@data
type Point
  int x, y
end type

let a = Point(1, 2)
let b = a.copy(y = 5)   // Point(1, 5)
a = b                    // structural equality
@sealed
Closes the type hierarchy. Only types declared in the same module may extend this type — the compiler knows every possible subtype.
@sealed
type Shape
  string kind
end type

type Circle extends Shape
  int radius
end type

type Rect extends Shape
  int w, h
end type
@singleton
Exactly one instance ever exists. The runtime creates it on first access and reuses it everywhere. No constructor is callable externally.
@singleton
type AppConfig
  string env
  int    maxConnections
end type

// accessed anywhere — same instance
AppConfig.env = "production"
@alias
Declares a named alias for an existing type. Interchangeable with the original — adds meaning without wrapping.
@alias UserId       = int
@alias EmailAddress = string

function find_user(UserId id)
  // id is an int — no wrapping overhead
end function
Functions
@callembed
The function body is substituted at every call site at compile time. No call overhead. Required when passing lambdas to avoid allocation.
@callembed
function clamp(int v, int lo, int hi)
  if v < lo
    return lo
  else if v > hi
    return hi
  else
    return v
  end if
end function

// no call generated — expands in place
let x = clamp(score, 0, 100)
@extends
Makes a free function callable as if it were a method on the named type. Does not modify the type — purely syntactic resolution.
@extends(string)
function truncate(string s, int max)
  if s.length <= max
    return s
  end if
  return s.slice(0, max) + "…"
end function

let t = title.truncate(40)
@operator
Maps a named function to an arithmetic or comparison operator for a given type. The function must take the type as its first argument.
@operator(+)
function add(Vector2 a, Vector2 b)
  return Vector2(a.x + b.x, a.y + b.y)
end function

// calls add() at compile time
let v = origin + offset
@receiver
The function runs in the context of the named type — its fields are directly in scope without qualification. Foundation for DSL builders.
@receiver(QueryBuilder)
function build_query()
  from("users")         // QueryBuilder.from
  where(active = true) // QueryBuilder.where
  limit(100)            // QueryBuilder.limit
end function
Safety & Contracts
@nullable
Marks a field or parameter as explicitly nullable. Without this annotation, the compiler assumes non-null and enforces it.
type Session
  string            token
  @nullable string  user
  @nullable int     expiresAt
end type

function greet(@nullable string name)
  if name = empty
    return "Hello, guest"
  end if
  return "Hello, " + name
end function
@throws
Declares that a function may raise an exception of the given type. Unlike Java, this is documentation only — callers are not forced to handle it.
@throws(IOError)
function read_file(string path)
  // ...
end function

@throws(AuthError, NetworkError)
function login(string user, string pass)
  // ...
end function
Architecture
@delegate
Implements a type's interface by forwarding all calls to a named field. Composition without boilerplate — the compiler generates every forwarding call.
type Logger
  function log(string msg)
end type

type TimestampedLogger
  @delegate Logger inner    // log() forwarded to inner.log() automatically
  string           format
end type

// caller uses TimestampedLogger.log() — no forwarding method written by hand
@async / @await
Marks a function as a coroutine. The function may be suspended and resumed without blocking the calling thread. Callers use @await to receive the result.
@async
function fetch(string url)
  let resp = @await http_get(url)
  return resp.body
end function

@async
function load_dashboard()
  let user    = @await fetch("/api/user")
  let reports = @await fetch("/api/reports")
  return Dashboard(user, reports)
end function
Deployment
@platform
Restricts or specialises a declaration to one or more compile targets. One codebase, per-target overrides where needed.
// shared declaration — compiles everywhere
function format_date(int timestamp) string

@platform(jvm)
function format_date(int timestamp)
  return java_format(timestamp, "yyyy-MM-dd")
end function

@platform(js)
function format_date(int timestamp)
  return js_date(timestamp).toISOString()
end function

@platform(c)
function format_date(int timestamp)
  return strftime("%Y-%m-%d", timestamp)
end function
