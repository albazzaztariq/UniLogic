JavaScript source translated into UL, with notes on where the languages diverge.

Example 1 — Timer
JavaScript
function Timer() {
    this.count = 0;
    var self = this;
    setInterval(function() {
        self.count++;
        if (self.count > 10) {
            self.stop();
        }
    }, 1000);
    this.stop = function() {
        var self = this;
        clearInterval(function() {
            self.count = 0;
        });
    };
}
UL
type Timer
  int count
end type

function tick(Timer <t>)
  t.count++
  if t.count > 10
    stop(t)
  end if
end function

function stop(Timer <t>)
  t.count = 0
end function

JavaScript's this does not refer to a fixed object — it rebinds depending on how a function is called. Inside a setInterval callback, this no longer refers to the Timer instance. The standard fix is var self = this, capturing the reference before entering the callback. This works, but it is defensive code written to work around the language, not to express intent.

In UL, there is no this. State belongs to the type, and functions that operate on that state receive it as an explicit pointer parameter — Timer <t>. There is no ambiguity about what t refers to, no closure capturing, and no rebinding. The structure is the same; the noise is gone.

tick and stop are plain functions that take a Timer pointer. The caller decides when to call them. The JavaScript version bundles the timer logic, the interval setup, and the stop implementation all into one constructor function — UL separates them cleanly.
