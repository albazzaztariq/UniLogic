XPile Compiler

XPile is the UniLogic compiler. It takes .ul source files through lexing, parsing, AST construction, and code generation. It currently emits four targets:

TargetOutputUse Case
CPortable C99Systems work, embedding, maximum portability
PythonPython 3Scripting, prototyping, interop with Python libraries
JavaScriptES6Browser and Node.js targets
LLVM IR.ll filesNative compilation via LLVM, optimization passes

The pipeline runs in stages: the lexer tokenizes the source, the parser builds an AST, and the codegen backend walks the AST to emit the selected target. Each target has its own backend module.

Usage

# Compile to C (default)
xpile main.ul

# Compile to Python
xpile main.ul --target python

# Compile to JavaScript
xpile main.ul --target js

# Emit LLVM IR
xpile main.ul --target llvm
Multi-target by design The same source compiles to any target without modification. Write once, deploy to systems code (C), scripts (Python), the browser (JS), or native binaries (LLVM).

Pipeline

.ul source → Lexer → Parser → AST → CodeGen → C | Python | JS | LLVM IR

REPL

The UniLogic REPL is an interactive shell for evaluating expressions and statements line-by-line. It runs through the same lexer and parser as XPile, so what works in the REPL works in compiled code.

Usage

# Start the REPL
ul

# Inside the REPL
ul> int x = 10
ul> print x * 2
20

The ul command launches the REPL directly. It supports variable declarations, expressions, function definitions, and control flow. State persists across lines within a session.

What the REPL is for

Testing syntax, exploring the type system, and verifying small logic before committing it to a file. It is not a debugger or a build tool.

VS Code Extension

The UniLogic VS Code extension provides editor integration for .ul files. It connects to the UniLogic language toolchain and surfaces information directly in the editor.

Features

FeatureDescription
Syntax HighlightingFull tokenization of UniLogic keywords, types, operators, literals, and comments
DiagnosticsInline error and warning markers from the compiler, updated on save
HoverType information and documentation on mouse hover for variables, functions, and types
Code WalkthroughStep-by-step annotations explaining what each section of code does
Diagnostics are compiler-backed Errors shown in the editor come from the same analysis pipeline as XPile. If the extension marks something red, the compiler will reject it too.

Code Walkthrough

The walkthrough feature annotates UniLogic source files with plain-language explanations. It is designed for reading unfamiliar code or onboarding onto an existing project. Annotations appear inline and can be toggled on or off.

UniLogic Project Manager

The Project Manager is a standalone GUI application for creating, organizing, and building UniLogic projects. It handles project structure, file management, and build configuration without requiring the command line.

Capabilities

FunctionDescription
Project creationScaffold new projects with the standard directory layout
File managementCreate, rename, move, and delete .ul source files within a project tree
Build settingsSelect compilation target (C, Python, JS, LLVM IR) and configure output paths
Build & runCompile and execute from the GUI with output displayed in an integrated console

When to use it

The Project Manager is useful for managing multi-file projects and for developers who prefer a graphical interface over the command line. It calls XPile under the hood — the compiler does the actual work.
