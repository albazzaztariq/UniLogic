Coding Agents like Claude Code and Codex are revolutionizing application development. However, there are lingering issues with code quality, even outright non-working code. UniLogic for Model and Agent Programming (UMAP) is a future project to design an intermediate representation between agent and user instead of only spoken words and the final code, closer to the actual code that it will be made of. Something close enough to natural language for anyone to use, but closer to programming syntax to drastically reduce interpretation and variability.

Industry Research & Developments

Title describes the relationship to this project. Link to full paper included with each entry.
#1

UIUC — Formal Query Language Placed Between Natural Language Prompt and Agent-Generated Code

Councilman, Fu, Gupta, Wang, Grove, Wang, Adve  ·  2025  ·  arxiv.org/abs/2507.13290

Natural language prompts to LLMs produce code that cannot be formally verified to be correct. The authors built Astrogator: a formally-defined query language placed between the user prompt and LLM output. The user expresses intent in that structured form; Astrogator verifies the generated code against it. The formal intermediary is what makes verification possible.

UMAP addresses the specification side of the same gap — the structured schema is the formal intermediary. Where Astrogator adds a verification layer on top of LLM output, UMAP starts from a representation that is already unambiguous. The two approaches are working the same problem from opposite ends.

Results
83% verification success for correct LLM-generated code. 92% detection rate for incorrect code. Tested on Ansible DSL tasks.
#2

FSE 2024 — LLM Code Translation Failures Traced to Semantic Divergence Between Source and Target Languages

Yang et al.  ·  FSE 2024  ·  ACM DOI:10.1145/3660778  ·  arXiv:2404.14646

Empirical study of LLM code translation across six language pairs. Three failure modes identified: LLM does not understand source logic (38.5%), missing I/O type information (14.9%), generated output diverges from source semantics without detection (41.4%). The dominant failure mode — undetected semantic divergence — accounts for the largest share of incorrect translations. UniTrans (test-case correction loops) was proposed as a workaround.

A single source representation that maps to any target without a translation step removes this failure mode entirely. The accuracy loss the study measures is the cost of the semantic gap between languages — the same gap a unified schema eliminates at the source.

Results
UniTrans improvements over baseline (avg. across 6 language pairs): LLaMA-7B +28.58% computational accuracy, +71.22% exact match. LLaMA-13B +19.20% / +36.42%. GPT-3.5 +4.02% / +13.28%. Smaller models benefited most — GPT-3.5 baseline was already 87.92% CA before correction.
#3

UT Austin — Survey: Leading Code Generation Systems Use Symbolic Program Structure as Backbone, Not End-to-End Neural

Chaudhuri  ·  Handbook on Neurosymbolic AI and Knowledge Graphs, IOS Press 2025  ·  cs.utexas.edu/~swarat/pubs/ns-handbook-2025.pdf

A UT Austin survey covering where AI-assisted programming research is heading. The finding: the most capable recent code generation systems — including Google's AlphaDev (which discovered faster sorting algorithms) and SWE-agent (automated software engineering) — don't rely on AI alone. They use structured, rule-defined code as a backbone, with AI handling search and generation on top of it. AI working without that structure produces results that are unreliable and impossible to verify.

Two problems the field hasn't solved: getting AI to generate program structures it hasn't seen before, and translating what a user wants into something precise enough for the system to act on correctly. The survey identifies the second as the key missing piece.

Relation to UMAP
Survey identified structured syntax as a core necessity. UMAP is built around the same idea.
#4

MIT CSAIL — Natural Language as Program Specification is Fundamentally Under-constrained

Solar-Lezama  ·  Program Synthesis Course  ·  MIT CSAIL  ·  Lecture 1  ·  Lecture 15

A graduate course — 25 lectures. Two are directly relevant. Lecture 1 frames the core problem: one of three named challenges is specification — how do you tell the system what you want? Natural language is the easiest way for a human to express intent but the most ambiguous for a system to act on. Any sentence has multiple programs that satisfy it with no reliable way to pick the right one. The alternative — formal mathematical specs — is precise but requires expertise most people don't have.

Lecture 15 covers the distinction between two agent architectures: one where the program is in control and the LLM is a tool it calls, and one where the LLM is in control and calls out to code when needed. The lecture treats these as the two fundamental models for how AI-assisted programming works.

Relation to UMAP
Lecture 1: a practical explanation of the problem space UMAP naturally fits as a solution. Lecture 15: UMAP is a program-in-control architecture. The schema runs the process; the LLM is an optional tool alongside it.
#5

OOPSLA 2024 — Formal Semantics Synthesised Automatically from Grammar and Interpreter

Liu, Murphy, Grover, Johnson, Reps, D'Antoni  ·  OOPSLA 2024  ·  dl.acm.org/doi/10.1145/3689724

Synantics takes a formal grammar and an executable interpreter and synthesises formal semantics expressed as Constrained-Horn Clauses, usable directly by verification and synthesis tools. The method applies only to languages that have a formal grammar and consistent interpreter — a prerequisite a schema-based language satisfies by construction.

Results
Formal semantics synthesised for 14 interpreters. Found and resolved an ambiguity in one language's interpreter; produced a 1.1× speedup in a downstream synthesiser.
#6 — Counter-position

arXiv 2024 — Prediction that Natural Language Will Become the Primary Programming Interface

Beheshti  ·  arXiv:2406.05409  ·  2024  ·  arxiv.org/abs/2406.05409

Predicts natural language will replace traditional code syntax as the primary programming interface, citing generative AI as the enabling technology. The accessibility argument is valid. The mechanism proposed — natural language in, correct executable software out — is what the entries above collectively address.

Relation to UMAP
Finance, telecom, and other sectors use codes, contracts, and other standardized and heavily minimized language for modification and quicker comprehension. Programming goes farther in the direction of requiring such notation.
